## Overview

This repository provides a structured approach to daily Linux and Bash practice. Each practice session is designed to take 30-60 minutes and focuses on building practical skills through hands-on exercises.

## Daily Practice Structure

### Daily Session Format (30-60 minutes)
1. **Warm-up (5-10 min)**: Review previous day's concepts
2. **New Concept (15-20 min)**: Learn new command/concept
3. **Hands-on Lab (20-30 min)**: Practice with real exercises
4. **Review & Document (5-10 min)**: Document what you learned

## Learning Path

### Week 1-2: Fundamentals
- Basic navigation and file operations
- File permissions and ownership
- Text viewing and manipulation
- Basic shell scripting

### Week 3-4: Intermediate
- Process management
- Package management
- User and group management
- Advanced file operations

### Week 5-6: Advanced
- Shell scripting (loops, conditionals)
- Text processing (sed, awk, grep)
- System monitoring and performance
- Networking basics

### Week 7-8: Expert
- Automation and cron jobs
- Bash functions and libraries
- Advanced text processing
- System administration tasks

## Weekly Schedule

### Monday: File System & Navigation
- Practice: `cd`, `ls`, `pwd`, `find`, `locate`
- Lab: Navigate complex directory structures

### Tuesday: File Operations
- Practice: `cp`, `mv`, `rm`, `mkdir`, `touch`, `cat`, `less`
- Lab: File management scenarios

### Wednesday: Text Processing
- Practice: `grep`, `sed`, `awk`, `cut`, `sort`, `uniq`
- Lab: Process log files and extract data

### Thursday: Permissions & Users
- Practice: `chmod`, `chown`, `chgrp`, `useradd`, `passwd`
- Lab: Set up proper file permissions

### Friday: Processes & System
- Practice: `ps`, `top`, `kill`, `systemctl`, `df`, `du`
- Lab: Monitor and manage system resources

### Weekend: Projects & Review
- Saturday: Work on integrated projects
- Sunday: Review week's learning and prepare notes
